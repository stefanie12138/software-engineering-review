二、简答题高频例题

1. 什么是软件危机？

  • 考点：SE2 什么是软件工程
  • 答题框架：定义 → 主要表现 → 产生原因
  • 真题直达：01-题目2：软件危机表现及原因

2. 简述软件生命周期各阶段任务。

  • 考点：SE2/SE3 软件过程模型
  • 答题框架：需求分析 → 概要设计 → 详细设计 → 编码实现 → 集成测试 → 确认测试
  • 真题直达：
  • 01-题目1：软件定义 + 生命周期各阶段工作
  • 01-题目3：生命周期各阶段内容
  • 01-题目4：生命周期各阶段任务

3. 软件工程三要素是什么？它们之间是什么关系？

  • 考点：SE2
  • 答题框架：过程、方法、工具；过程定义活动顺序，方法提供技术手段，工具支持实现
  • 真题直达：01-题目6：软件工程定义、三要素及主要目标

4. 简述五种基本软件过程模型及其适用场景。

  • 考点：SE3.1
  • 答题框架：瀑布、增量、迭代、原型、螺旋；从指导思想、特点、适用场景三个维度列表对比
  • 真题直达：
  • 01-题目7：五种过程模型对比
  • 02-题目5：建模方法分类及特点

5. 什么是需求工程？需求工程包括哪些活动？

  • 考点：SE4
  • 答题框架：需求工程定义 → 获取 → 分析 → 文档化 → 确认验证 → 管理
  • 解析直达：00-全书重点速查：需求工程活动

6. 需求获取有哪些常用方法？

  • 考点：SE5
  • 答题框架：访谈、问卷、会议、现场观摩、业务资料分析、原型、头脑风暴等
  • 解析直达：00-全书重点速查：SE5 获取软件需求

7. 简述顺序图与通信图（协作图）的区别。

  • 考点：SE6 UML 交互图
  • 答题框架：同属 UML 交互图、语义等价、可相互转换；顺序图强调时间顺序，通信图强调对象结构与合作关系
  • 真题直达：
  • 02-题目1：顺序图与协作图定义及区别
  • 01-题目8：协作图与顺序图区别

8. 什么是模块独立性？如何度量？

  • 考点：SE7 软件设计基础
  • 答题框架：内聚度（越高越好）+ 耦合度（越低越好）；设计原则为高内聚、低耦合
  • 真题直达：03-题目3：模块独立性两个度量概念及设计原则

9. 面向对象设计 6 大原则是什么？

  • 考点：SE7
  • 答题框架：SOLID + 迪米特法则
  • 真题直达：03-题目4：面向对象设计6大原则

10. 三种软件设计方法及其特点？

  • 考点：SE7
  • 答题框架：结构化、面向对象、基于构件/重用
  • 真题直达：03-题目1：三种软件设计方法及其特点

11. 黑盒测试与白盒测试有什么区别？

  • 考点：SE13 软件测试
  • 答题框架：从依据、是否需了解代码、测试对象、典型方法、典型阶段等维度列表对比
  • 真题直达：
  • 04-题目5 第2问：黑盒/白盒区别
  • 02-题目6 第2问：黑盒/白盒区别

12. 简述软件测试的层次。

  • 考点：SE13
  • 答题框架：单元测试 → 集成测试 → 确认测试 → 系统测试 → 回归测试
  • 真题直达：
  • 04-题目1：测试流程及每个阶段测试对象
  • 04-题目2：测试步骤及每个阶段测试对象
  • 04-题目3：软件测试步骤及测试对象

三、设计题综合例题

设计题共 60 分，是考试核心。以下按图类型整理常考例题与解题入口，链接直接定位到具体题目。

3.1 用例图

真题来源：

  • 02-题目3 第1问：医院预约挂号管理系统用例图
  • 03-题目7 第1问：医院预约挂号管理系统用例图
  • 06-示例：图书馆管理系统用例图
  • 06-综合练习1：医院预约挂号系统用例图

解题步骤：

  1. 找 Actor
  2. 找 Use Case（动宾结构）
  3. 画系统边界
  4. 建立关联 / <<include>> / <<extend>>

3.2 顺序图 / 通信图

真题来源：

  • 02-题目3 第2问：医院预约挂号顺序图
  • 03-题目7 第2问：医院预约挂号顺序图
  • 06-示例：ATM 取款顺序图

解题步骤：

  1. 确定参与对象（Actor → 边界类 → 控制类 → 实体类 → 外部接口）
  2. 画生命线
  3. 按时间顺序画消息（同步/异步/返回）
  4. 标激活条、alt / opt / loop 片段

3.3 类图

真题来源：

  • 06-示例：学生选课系统类图
  • 06-示例：继承（泛化）关系
  • 06-综合练习3：博客系统类图

解题步骤：

  1. 找名词 → 候选类
  2. 筛选有效类
  3. 确定属性与方法
  4. 确定关系（关联/聚合/组合/泛化）并标多重性

3.4 状态图

真题来源：

  • 06-示例：订单生命周期状态图

解题步骤：

  1. 找对象生命周期
  2. 列状态
  3. 找触发事件和监护条件
  4. 画初始/终止状态

3.5 活动图

真题来源：

  • 06-示例：请假审批活动图

解题步骤：

  1. 找活动
  2. 定顺序与分支
  3. 画泳道
  4. 标 start / end

3.6 数据流图（DFD）

真题来源：

  • 02-题目2：发帖子系统 DFD
  • 02-题目4：CCMS 信用卡管理系统 DFD
  • 03-题目5：发帖子系统 DFD
  • 03-题目6：CCMS 信用卡管理系统 DFD
  • 06-示例：在线订餐系统顶层/0层/1层 DFD
  • 06-综合练习2：快递包裹查询系统 DFD

解题步骤：

  1. 画顶层图（上下文图）：1 个加工 0 + 外部实体
  2. 画 0 层图：分解为 1、2、3… 加工 + 数据存储
  3. 画 1 层图：细化 0 层加工为 1.1、1.2…
  4. 检查平衡原则

3.7 DFD → 软件结构图

真题来源：

  • 02-题目2：发帖子系统 DFD → 结构图
  • 02-题目4：CCMS DFD → 结构图
  • 03-题目5：发帖子系统 DFD → 结构图
  • 03-题目6：CCMS DFD → 结构图
  • 06-教程：DFD → 软件结构图

解题步骤：

  1. 判断 DFD 类型：变换型 or 事务型
  2. 变换型：确定输入流 / 变换中心 / 输出流 → 一级分解 → 二级分解
  3. 事务型：确定事务中心 / 接收路径 / 动作路径 → 一级分解 → 处理每条路径
  4. 优化模块结构

3.8 程序流程图 + 环形复杂度 + 路径覆盖

真题来源：

  • 03-题目2：判断三角形流程图 + 环形复杂度 + 路径覆盖
  • 04-题目4：判断三角形流程图 + 环形复杂度 + 路径覆盖
  • 06-教程：程序流程图 + 环形复杂度 + 路径覆盖

解题步骤：

  1. 画程序流程图
  2. 转换为控制流图
  3. 计算 $V(G) = E - N + 2$ 或 $V(G) = P + 1$
  4. 列出基本路径
  5. 为每条路径设计测试用例

3.9 等价类划分 + 边界值分析

真题来源：

  • 02-题目6：报表日期等价类 + 边界值 + 黑盒/白盒区别
  • 03-题目8：报表日期等价类 + 边界值 + 黑盒/白盒区别
  • 04-题目5：报表日期等价类 + 边界值 + 黑盒/白盒区别
  • 06-教程：黑盒测试用例设计

解题步骤：

  1. 划分有效等价类 / 无效等价类
  2. 边界值分析（范围端点、次大/次小）
  3. 设计测试用例（输入、所属等价类、预期输出）

3.10 N-S 图 / 控制流图

真题来源：

  • 06-示例：判断三角形 N-S 图
  • 06-示例：简单函数控制流图
